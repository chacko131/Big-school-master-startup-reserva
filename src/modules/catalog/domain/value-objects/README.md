# `catalog/domain/value-objects`

Contiene los value objects del dominio del módulo `catalog`.

## Qué son aquí

Son objetos que representan un valor del negocio y se preocupan de mantenerse válidos.

No tienen identidad propia como una entidad, pero sí reglas internas.

## Ejemplos de uso

- un slug normalizado
- una hora válida
- una cantidad válida

## Por qué son útiles

- evitan repetir validaciones
- encapsulan reglas pequeñas
- hacen el dominio más claro y expresivo

## Estado actual

La carpeta está preparada para los objetos de valor del catálogo cuando haga falta modelarlos.
